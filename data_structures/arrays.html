Basics of Arrays

An array (in general) is a linear structure that has a fixed size at the time of construction. The elements are placed contiguously in memory, meaning that it is near in memory but does not necessarily touch the memory. Inspection and update of some element in the array takes only \(O(1)\) constant time. More about the performance of array operations can be found here.

In Java, arrays are homogeneous. The components (or the elements of the array) must be all of the same type. And Object[] arrays can contain Objects of any subtype. Hence saying Object[] stuffs = {6, "ball", true, 'q', p} is valid (where Point p = new Point(2, 3) has already been declared before the creation of the array.

Also in Java, we say that an array is an Object. Hence, we allocate dynamically by the keyword new. And this is why we have been creating and declaring and initializing arrays this way: int[] nums = new int[10];.

The initial values of an array in Java depends on the type of elements it carries. For instance, an int[] array holds all 0 values initially (and of course 0.0 for the double[] type). If the array was supposed to boolean[] values, then the initial values would all have been false. And for Object[] arrays, then the initial values have to be null.

We can access the value of the elements within the array through their indices. If an array \(A\) has \(n\) elements, then the range of what we can access in \(A\) is from \(A[0...n-1]\). And this we can do by \(A[0], A[1], ..., A[n-1]\). In order to determine how many elements the array \(A\) has, simply type up A.length, and it should return to you \(n\).

Common Mistakes in using Arrays
  • Getting the size of an array uses the .length instance (or attribute). It is not a method.
    char[] A = {'a', 'b', 'c', 'd'};
for (int i = 0; i < A.length(); i++) {   // wrong, it should be A.length
  // do something here
}
  • The array is unallocated. The Java code below is incorrect because we have created an array and yet we have not allocated how many elements the array will have. Hence the program does not know how much memory to allocate for the array.
    int[] numbers;
numbers[0] = 15;                        // wrong, unallocated array
  • Attempting to access an index that is out of bounds. Note that if we have an array \(A\) where A.length = n, then we can only access elements from the range \(A[0...n-1]\). Attempting to access a negative index or an index larger than \(n-1\) gives an ArrayIndexOutOfBoundsException.
    int[] nums = {10, 20, 30, 40, 50};
nums[5] = 60;                           // wrong, you can only access nums[0...4]
System.out.println(nums[-1])            // wrong, negative index does not exist
Note: These mistakes are more of runtime and syntactic that produces exceptions. There are still more mistakes (the logical ones) that will be discussed further in the next sections to come.

2D Arrays in Java
They are usually called a matrix or a grid and can be implemented as 2D arrays. This means that 2D arrays are simply arrays made up of arrays. (Note that the size, specified by dimensions, of a 2D array is known to be by row and by column. For instance, we wish to declare a 2D array that has 3 rows and 4 columns. Then this means that our "main" array will have a length of 3 and each element in this "main" array contains another array of length 4 this time. Below shows a code of two ways to implement such. 

class MatrixImplemeneter {
  public static void main (String[] args) {
    int rows = 3;
    int columns = 4;

      // first way: one line direct way
    int[][] nums1 = new int[rows][columns];

      // second way: using a loop
    int[][] nums2 = new int[rows][];
    for (int i = 0; i < rows; i++) 
      nums2[i] = new int[columns];
  }
}

The Utility Library for Arrays
The java.util.Arrays class in Java has useful in built-in methods:
  1. equals(A,B) - returns true if \(A\) and \(B\) have an equal number of elements and every corresponding pair of elements in the two arrays are equal
  2. fill(A,x) - store \(x\) into every cell of \(A\)
  3. sort(A)
  4. binarySearch(int[] A, int key) - uses the binary search algorithm to find key's index location in \(A\) and returns -1 if \(key \notin A\); assumes that \(A\) is sorted as a precondition

Arrays as Pointers

The name of the array is a reference (also called pointer) to the place in memory where the array is stored. The reference to an object holds the address of the actual object. Here is an example to illustrate: suppose you have some array \(A = [15, 29, 47, 63, 98, 84, 31, 65, 80, 11]\). And you decided to copy this array to array \(B[]\). 

int[] A = {15, 29, 47, 63, 98, 84, 31, 65, 80, 11};
int[] B = A;

The code above shows the array copy of \(A\) to \(B\) and the above picture shows the image visual of what is happening in array copy. Notice how \(A\) points to the array (as we know that array is an Object and acts as a pointer reference). But then also, notice how \(B\) also points to the same array as \(A\). So we know from here that the array copy not only copies the elements (and their respective corresponding positions) from \(A\) to \(B\) but also their memory addresses. Let us verify this by changing one of the elements of \(B\) and we will see that the change also takes in effect in \(A\). Without loss of generality, we can do the same when we change one of the elements in \(A\) and see the change reflected upon \(B\) as well. 

B[3] = 20;
System.out.println(A[3]);     // prints 20
System.out.println(B[3]);     // prints 20


Array cloning on the other hand is slightly similar to array copying. The only difference is that upon cloning, the new array that was created from the clone receives a new memory location different from the array that was cloned. To illustrate what I mean, see the code below and its corresponding visual image that depicts the code. 

int[] A = {15, 29, 47, 63, 98, 84, 31, 65, 80, 11};
int[] B = A.clone();          // the array clone method

So here, we see that a new instance of an array was created. As a result, upon updating the element in \(B[3]\), this leaves the element \(A[3]\) untouched - compared to the array copy version that changes \(A[3]\) as well. Below shows the update code and the corresponding image. 

B[3] = 20;
System.out.println(A[3]);     // still should print 63
System.out.println(B[3]);     // now prints 20


Performance and Limitations

As mentioned earlier, the access and update (of a single element) methods for arrays is constant time \(O(1)\). But what about the other methods? Take for instance insertion. Insertion into an array, regardless of whether or not it is full takes linear \(O(n)\) time. Remember that by instantiating an array, regardless of what type is it, initial values have already been given into every cell of the array. So technically, we can say that the array is "already full". And by inserting an element into an array, we are adding an element into the array. This entails re-creating another array that has length 1 more than the length of the original array, and depending on where you wish to add the element (either at the beginning, end or middle), \(O(n)\) time is still needed as elements get transferted from the original to the new array. Deletion works the same way, removing an element from the array, entails checking first if the element is in the array (the library may have a built-in method for it), but the point is upon checking for the existence of the element, you still have to re-size the array. You just cannot leave an empty hole in your array after deleting an item from the array.

So you see, the arrays have its own limitations. One main limitation it has is its static data structure. If you have an array of length \(n\) and you needed to store \(k\) items in this array, but \(k > n\), then we run out of memory. To fix this, we can use a bigger array size than expected before we even try storing the items. So for instance you can create an array that can contain 1000 elements, and then use that to store the ID numbers of students of a particular class. But notice that if you have a small class, then that leads to a wastage of memory. So that isn't a good fix at all, since it lead to another problem. What if you only create the array once you find out how many students you have in the class? But the problem of this fix is that this takes extra runtime. So even if you say "I'll wait until after all my students have enrolled in my course before I can create my student ID array." Again, a good idea since you would not have to run out memory nor have memory wastage since you will know how many students have enrolled in the class. But if several students start coming in still, then you would have to resize your array every single time a student enrolls for the course. This has a similar problem for students that would drop the course as you would have to take them out of the list and need to delete their student IDs from the array. So insertion/deletion is time consuming due to shifting issues. And finally, the time complexity of a growing array also needs to be considered.

  2. Matrix Spiral. Implement an algorithm that would fill some \(n \times n\) matrix with integers from \(0\) to \(n^2-1\), starting from the center, spiraling until the entire 2D matrix is filled.

    Input Format: In a single line of input separated by single spaces,
    • The first input is the number of rows/columns, \(n\).

    • The third input indicates the matrix's spiral direction ('C' for clockwise or 'W' for counter-clockwise)

    Constraints:
    • \(1 \leq n \leq 1000\)

    Output Format: Print the 2D matrix (adjacent horizontal cells need to be separated by a single space). You do not need to worry about the alignment.

    Sample Input:
    3 R C

    Sample Output:
    6 7 8
    5 0 1
    4 3 2
